# Votely — Google スプレッドシートで動く匿名オンライン投票システム

Votely は、**Googleスプレッドシート + Google Apps Script + GitHub Pages** を組み合わせた、無料で使える匿名オンライン投票システムです。

- 投票者は**専用URL**をクリックするだけで投票できます（アプリのインストール不要）
- 投票結果は**匿名**で処理されます（誰が何に投票したかは記録されません）
- **どのGoogleアカウントでログインしていても**同じURLで投票できます
- 複数の投票項目にも対応しています

---

## 必要なもの

- Google アカウント（Gmail があれば OK）
- GitHub アカウント（無料プランで OK）

---

## 動作モードについて

Votely には2つの動作モードがあり、投票規模に応じて切り替えられます：

### 🐢 通常モード（デフォルト）
- **対象**: 小〜中規模の投票（〜100人程度）
- **動作**: 投票時にスプレッドシートに直接書き込み
- **特徴**: シンプルで安定、リアルタイム反映
- **推奨**: 一般的な会議、小規模な投票

### 🚀 高速モード
- **対象**: 大規模一斉投票（100人以上）
- **動作**: 投票データを一時キャッシュに保存 → バッチ処理で一括書き込み
- **特徴**: Googleスプレッドシートの書き込み制限を回避、同時投票対応
- **推奨**: 大規模な選挙、一斉投票が必要な場合

**モード切り替え方法**: 設定シートのA6セルで「通常モード」または「高速モード」を選択

## セットアップ手順

### ステップ 1 — このリポジトリをコピーする

1. このページ上部の緑色の **「Use this template」** ボタンをクリックします
2. 「Create a new repository」を選択します
3. Repository name（リポジトリ名）に好きな名前を入力します（例: `votely`）
4. **Public** を選択します（GitHub Pages を無料で使うために必要）
5. **「Create repository」** をクリックします

> 自分専用のコピーが作成されます。以降の作業はすべて自分のリポジトリで行います。

---

### ステップ 2 — Googleスプレッドシートを準備する

1. [Google スプレッドシート](https://sheets.google.com) を開き、**新しいスプレッドシートを作成**します
2. スプレッドシートに名前を付けます（例: `Votely_投票名`）

> **注意**: シートの手動作成は不要です。次のステップでGASコードを貼り付けた後、自動構築機能で必要なシートが作成されます。

---

### ステップ 3 — Google Apps Script にコードを貼り付ける

1. 先ほどのスプレッドシートで、上部メニューの **「拡張機能」→「Apps Script」** をクリックします
2. 左側のファイル一覧に「コード.gs」があります。その中身を**すべて削除**します
3. このリポジトリの [`コード.gs`](コード.gs) を開き、内容を**すべてコピー**します
4. GASエディタに**貼り付けて**、左上の**保存ボタン（フロッピーアイコン）**をクリックします
5. 左側のファイル一覧で **`appsscript.json`** を開き、このリポジトリの [`appsscript.json`](appsscript.json) の内容を**すべてコピーして貼り付け**、保存します
6. スプレッドシートに戻り、上部メニューに **「Votely 管理」** が追加されていることを確認します

---

### ステップ 4 — GAS をデプロイしてURLを取得する

1. GASエディタ右上の **「デプロイ」→「新しいデプロイ」** をクリックします
2. 左上の歯車アイコンをクリックし、**「ウェブアプリ」** を選択します
3. 以下のように設定します：
   - 説明: `Votely`（任意）
   - **次のユーザーとして実行: 自分**
   - **アクセスできるユーザー: 全員**
4. **「デプロイ」** をクリックします（Googleアカウントの権限確認が出たら「許可」）
5. 表示された **ウェブアプリのURL** をコピーします
   - 例: `https://script.google.com/macros/s/XXXXXXXXXXXXXXXX/exec`
6. スプレッドシートの **「設定」シートの A4セル** にこのURLを貼り付けます

---

### ステップ 5 — GitHub Pages を有効化してURLを取得する

1. 自分のGitHubリポジトリを開きます
2. 上部の **「Settings」** タブをクリックします
3. 左メニューの **「Pages」** をクリックします
4. **「Source」** を「Deploy from a branch」に設定します
5. Branch を **「main」**、フォルダを **「/docs」** に設定して **「Save」** をクリックします
6. 数分後にページを更新すると、**GitHub Pages のURL** が表示されます
   - 例: `https://あなたのユーザー名.github.io/リポジトリ名`
7. スプレッドシートの **「設定」シートの A5セル** にこのURLを貼り付けます（末尾のスラッシュは不要）

---

### ステップ 6 — config.js を編集する

1. 自分のGitHubリポジトリで **`docs/config.js`** を開きます
2. 右上の鉛筆アイコン（Edit）をクリックします
3. 以下のように書き換えます：

```js
var GAS_URL = 'ステップ4でコピーしたURL';
```

例:
```js
var GAS_URL = 'https://script.google.com/macros/s/XXXXXXXXXXXXXXXX/exec';
```

4. 下部の **「Commit changes」** をクリックして保存します

---

### ステップ 7 — データベース自動構築

1. スプレッドシートに戻り、上部メニューの **「Votely 管理」→「⓪ 【初回のみ】データベース自動構築」** を実行します
2. これで以下の4つのシートが自動的に作成・設定されます：

| シート名 | 用途 | 自動設定内容 |
|----------|------|--------------|
| `設定` | 投票の設定情報 | A1〜A6セルに説明コメント、B列に投票例を設定 |
| `名簿とトークン` | 投票者の管理 | ヘッダー行を設定、列幅調整 |
| `投票箱` | 匿名の投票データ | ヘッダー行を設定、タイムスタンプ列を含む |
| `システム管理` | 集計完了フラグ | 集計状態を管理するフラグを設定 |

3. **「設定」シート**に以下を入力します（A6セルには自動で「通常モード」が設定されます）：

| セル | 入力内容 | 例 | 備考 |
|------|----------|----|------|
| A1 | 主催者のメールアドレス（複数の場合はカンマ区切り。2件目以降はBCCで送信） | `you@example.com,sub@example.com` | 【必須】結果通知先 |
| A2 | 投票の締め切り日時 | `2026/06/01 18:00` | 【任意】形式: `YYYY/MM/DD HH:MM` |
| A3 | 事務局パスワード（当日参加者登録に使用） | `password123` | 【必須】admin.htmlで使用 |
| A4 | GASのURL（ステップ4で取得します） | | 【必須】後で設定 |
| A5 | GitHub PagesのURL（ステップ5で取得します） | | 【必須】後で設定 |
| **A6** | **動作モード** | `通常モード` | 【自動設定】`通常モード` または `高速モード`に変更可能 |
| B1 | 投票1のタイトル | `議長選出` | 投票項目のタイトル |
| B2〜 | 投票1の選択肢（各行に1つ） | `候補A`、`候補B` | 選択肢を縦に並べる |
| C1 | 投票2のタイトル（3つ以上も同様にD、E列と続けます） | `予算案` | 追加投票項目 |
| C2〜 | 投票2の選択肢（各行に1つ） | `賛成`、`反対` | 選択肢を縦に並べる |

### ステップ 8 — 動作確認

1. ブラウザで `https://あなたのユーザー名.github.io/リポジトリ名/index.html?token=test` を開きます
2. 「有効なトークンが見つかりません」というエラーが表示されれば、GASとの通信は成功しています

> **注意**: データベース自動構築を実行する前にGAS URLとGitHub Pages URLを設定シートに入力する必要はありません。自動構築後にA4、A5セルにURLを入力してください。

## 投票の実施方法

### 事前準備（投票日の前）

1. スプレッドシートの **「名簿とトークン」シート** のA列（2行目以降）に投票者のメールアドレスを入力します
2. **【重要】動作モードの確認**: 設定シートのA6セルで適切なモードを選択
   - 小規模投票: `通常モード`
   - 大規模一斉投票: `高速モード`
3. **高速モードを使用する場合**:
   - 投票開始前に `「Votely 管理」→「③ ▶ 投票開始（※高速モード用: バッチ処理開始）」` を実行
   - これで1分間隔のバッチ処理が開始されます
4. スプレッドシート上部メニューの **「Votely 管理」→「① トークン生成＋案内メール送信」** を実行します
5. 各投票者に専用URLが記載されたメールが送信されます

### 当日（当日参加者がいる場合）

1. `https://あなたのユーザー名.github.io/リポジトリ名/admin.html` を事務局のPCで開きます
2. 事務局パスワード（設定シートA3）を入力して認証します
3. 当日参加者のメールアドレスを入力して投票URLを発行し、参加者に伝えます

### 締め切り後（集計）

- **高速モードを使用した場合**:
  - 投票終了後に `「Votely 管理」→「④ ⏹ 投票終了（※高速モード用: バッチ処理停止）」` を実行
  - 残りの投票データをスプレッドシートに書き込みます
- **自動集計**: `「Votely 管理」→「⑥ 締め切りトリガーをセットアップ」` を事前に実行しておくと、締め切り5分後に自動で集計・メール送信されます
- **手動集計**: `「Votely 管理」→「⑤ 集計・結果通知（手動実行）」` を実行します

## ファイル構成

```
リポジトリ/
├── コード.gs          # Google Apps Script のコード（GASに貼り付けて使う）
│                     # 通常モード/高速モードの切り替え機能を含む
├── appsscript.json    # GAS の設定ファイル（変更不要）
├── README.md          # このファイル（セットアップ手順とモード解説）
├── index.html         # 投票画面（ルートディレクトリ）
└── docs/              # GitHub Pages で公開されるフォルダ
    ├── index.html     投票画面
    ├── admin.html     事務局ポータル（当日参加者登録）
    ├── config.js      GASのURLを設定するファイル（★ステップ6で編集）
    └── .nojekyll      GitHub Pages の設定ファイル（変更不要）
```

## よくある質問

**Q. 投票は本当に匿名ですか？**  
A. はい。「投票箱」シートには投票日時と選択内容だけが記録され、メールアドレスやトークンは一切記録されません。誰が何に投票したかを後から追跡することは設計上できません。

**Q. 同じURLで2回投票できますか？**  
A. できません。一度投票すると「投票済み」フラグが立ち、同じURLでの再投票は拒否されます。

**Q. 投票項目を3つ以上にできますか？**  
A. できます。設定シートのC列、D列、E列…と続けて入力するだけで自動的に対応します。

**Q. 無料で使えますか？**  
A. Google アカウントと GitHub の無料プランの範囲内で使えます。ただし GAS のメール送信には1日あたりの上限（無料アカウントで100件/日）があります。

**Q. 通常モードと高速モードの違いは？**  
A. **通常モード**は投票時に直接スプレッドシートに書き込むシンプルな方式です。**高速モード**は大規模投票向けで、投票データを一時キャッシュに保存し、バッチ処理で一括書き込みします。これによりGoogleスプレッドシートの同時書き込み制限を回避できます。

**Q. 高速モードはいつ使うべきですか？**  
A. 100人以上の同時投票が予想される場合や、投票開始時間が決まっている大規模な選挙で使用してください。小規模な会議では通常モードで十分です。

**Q. 高速モードで投票データが失われることはありませんか？**  
A. いいえ。高速モードでは投票データがGoogleのCacheServiceに最大6時間保存され、1分間隔のバッチ処理でスプレッドシートに書き込まれます。投票終了時に「投票終了」を実行すれば、残りのデータも確実に書き込まれます。

**Q. モードは途中で切り替えられますか？**  
A. 投票開始前であれば自由に切り替えられます。投票中に切り替えると、処理方式が変わるためデータの整合性に問題が生じる可能性があります。投票開始前に適切なモードを選択してください。

**Q. シートを手動で作成する必要はありますか？**  
A. いいえ、手動でシートを作成する必要はありません。「Votely 管理」→「⓪ 【初回のみ】データベース自動構築」を実行すると、必要な4つのシートが自動的に作成され、適切な設定が行われます。

**Q. 自動構築を実行した後、設定を変更できますか？**  
A. はい、自動構築後に設定シートの内容を自由に変更できます。ただし、シート名は変更しないでください。シート名を変更するとシステムが正常に動作しなくなります。
